# 文件命名规范与编码标准

## 1. 文件命名规范

### 1.1 通用文件命名规则
- **使用小写字母**：所有文件名使用小写字母
- **使用下划线分隔**：单词之间使用下划线 `_` 分隔，避免使用空格或连字符
- **描述性命名**：文件名应清晰描述文件的功能或内容
- **避免特殊字符**：不使用除下划线以外的特殊字符
- **文件扩展名**：使用适当的文件扩展名，如 `.py`、`.md`、`.json` 等

### 1.2 模块文件命名
- **功能模块**：`feature_module.py`
- **工具模块**：`utils.py`、`helpers.py`
- **配置文件**：`config.py`、`settings.py`
- **数据处理**：`data_processor.py`、`data_loader.py`
- **模型文件**：`model.py`、`predictor.py`

### 1.3 测试文件命名
- **单元测试**：`test_module.py`
- **集成测试**：`test_integration.py`
- **功能测试**：`test_functionality.py`

### 1.4 脚本文件命名
- **日常运行脚本**：`run_daily.py`
- **数据获取脚本**：`fetch_data.py`
- **分析脚本**：`analyze_data.py`

### 1.5 文档文件命名
- **说明文档**：`README.md`
- **使用手册**：`MANUAL.md`
- **技术文档**：`docs/` 目录下的相关文档

## 2. 编码标准

### 2.1 缩进与格式
- **缩进**：使用4个空格进行缩进，不使用制表符
- **行宽**：每行代码不超过80个字符
- **空行**：
  - 函数与函数之间使用两个空行
  - 类与类之间使用两个空行
  - 类内部方法之间使用一个空行
  - 逻辑块之间使用适当的空行

### 2.2 命名规范
- **变量名**：使用小写字母和下划线，如 `user_name`
- **函数名**：使用小写字母和下划线，如 `calculate_average()`
- **类名**：使用驼峰命名法，如 `DataProcessor`
- **常量**：使用全大写字母和下划线，如 `MAX_ITERATIONS`
- **模块名**：使用小写字母和下划线，如 `data_utils`

### 2.3 注释规范
- **文件头部注释**：每个文件顶部添加文件说明、作者、创建日期等信息
- **函数注释**：使用文档字符串（docstring）描述函数功能、参数、返回值
- **代码注释**：对复杂逻辑或关键算法添加注释
- **注释格式**：使用 `#` 进行单行注释，多行注释使用三引号 `"""`

### 2.4 代码风格
- **导入语句**：
  - 标准库导入放在最前面
  - 第三方库导入放在中间
  - 本地模块导入放在最后
  - 每组导入之间使用空行分隔
- **空格使用**：
  - 运算符两侧使用空格，如 `a = b + c`
  - 逗号后使用空格，如 `func(a, b, c)`
  - 冒号后使用空格，如 `if x > 0:`
- **括号使用**：
  - 函数调用和定义时，括号内不使用空格
  - 列表、字典、元组等括号内不使用空格

### 2.5 异常处理
- **异常捕获**：使用 `try-except` 块捕获异常
- **异常类型**：尽量捕获具体的异常类型，避免使用 `except Exception`
- **异常处理**：对异常进行适当的处理，如记录日志或返回错误信息

### 2.6 代码质量
- **代码复用**：避免重复代码，提取公共功能为函数或类
- **模块化**：将功能分解为小的、可管理的模块
- **可读性**：优先考虑代码可读性，避免过度优化
- **测试覆盖**：为关键功能编写单元测试

## 7. 代码示例

### 7.1 函数定义示例
```python
def calculate_moving_average(data, window):
    """
    计算移动平均值
    
    Args:
        data (list): 数据列表
        window (int): 窗口大小
    
    Returns:
        list: 移动平均值列表
    """
    if window <= 0:
        raise ValueError("窗口大小必须大于0")
    
    result = []
    for i in range(len(data) - window + 1):
        window_data = data[i:i+window]
        average = sum(window_data) / window
        result.append(average)
    
    return result
```
