# 推送系统数据真实性问题整改计划

## 问题确认

经过代码审查，**确认用户反馈的所有问题均存在**，问题根源是推送系统使用了大量硬编码和模拟数据，而非从真实数据源获取。

### 问题定位

| 问题类型 | 代码位置 | 具体问题 |
|---------|---------|---------|
| 市场数据硬编码 | `a_stock_push_v4.py:206-213` | 上证、深证、创业板指数点位和涨跌幅全部硬编码 |
| 因子数据模拟 | `a_stock_push_v4.py:56-175` | `_generate_simulated_factors` 方法随机生成PE/PB/ROE等 |
| 股票选择硬编码 | `a_stock_push_v4.py:61-69` | 7只股票硬编码选定，非量化选股结果 |
| 市值数据伪造 | `a_stock_push_v4.py:146` | `market_cap = np.random.uniform(500, 20000)` 随机生成 |
| 行业偏离度硬编码 | `a_stock_push_v4.py:266-271` | 行业权重数据完全编造 |
| 风险指标硬编码 | `a_stock_push_v4.py:224-236` | 波动率、VaR、最大回撤全部硬编码 |
| 个股波动率硬编码 | `a_stock_push_v4.py:324` | 所有个股波动率统一硬编码为15.2% |
| 历史决策编造 | `a_stock_push_v4.py:563-570` | 历史决策数据硬编码 |

---

## 整改方案

### 第一阶段：数据源对接（核心）

#### 任务1：创建真实市场数据获取模块
- 新建 `code/market_data_fetcher.py`
- 实现功能：
  - 获取实时指数数据（上证、深证、创业板）
  - 获取市场情绪数据（涨跌停数量、成交量、北向资金）
  - 获取行业板块涨跌数据

#### 任务2：创建真实个股数据获取模块
- 新建 `code/stock_data_fetcher.py`
- 实现功能：
  - 获取个股实时价格
  - 获取个股财务指标（PE、PB、ROE、市值等）
  - 获取个股行业分类

#### 任务3：对接现有α选股器
- 修改 `a_stock_push_v4.py`
- 使用 `alpha_stock_selector.py` 的真实选股结果
- 移除 `_generate_simulated_factors` 方法

### 第二阶段：风险指标真实计算

#### 任务4：实现真实波动率计算
- 创建 `code/risk_calculator.py`
- 基于历史价格计算：
  - 组合波动率（20日年化）
  - 个股波动率
  - Beta值（相对沪深300）

#### 任务5：实现真实VaR计算
- 在 `risk_calculator.py` 中添加：
  - 历史模拟法VaR
  - 参数法VaR

#### 任务6：实现真实回撤计算
- 在 `risk_calculator.py` 中添加：
  - 当前回撤计算
  - 最大回撤计算

### 第三阶段：行业配置真实计算

#### 任务7：实现行业权重计算
- 创建 `code/industry_analyzer.py`
- 基于持仓计算：
  - 当前行业权重
  - 行业偏离度
  - 行业集中度

### 第四阶段：历史决策真实记录

#### 任务8：对接持仓跟踪系统
- 使用 `portfolio_tracker.py` 的真实持仓数据
- 读取 `data/trading_decisions.json` 的真实决策记录
- 移除硬编码的历史决策

### 第五阶段：推送系统重构

#### 任务9：重构推送内容生成
- 修改 `generate_push_content` 方法
- 所有数据从真实模块获取
- 添加数据获取失败的降级处理

#### 任务10：添加数据验证层
- 创建 `code/data_validator.py`
- 验证规则：
  - 指数点位合理性（上证3000-6000点）
  - 成交量合理性（万亿级别）
  - PE/PB合理性（正值、非极端值）
  - 市值合理性（与实际相符）

## 风险评估

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 数据源不可用 | 推送失败 | 添加降级逻辑，使用缓存数据 |
| 数据获取延迟 | 推送延迟 | 设置超时，异步获取 |
| 数据格式变化 | 解析失败 | 添加异常处理和格式验证 |
